# DolanClaw 完整使用指南

欢迎使用 DolanClaw！这是一个强大的本地 Agentic AI 编程平台。与普通的对话式 AI 不同，DolanClaw 具备**自主执行能力**，可以在您的本地计算机上读写文件、运行命令、搜索代码，并自动规划多步任务。

本文档将为您详细介绍平台的所有功能、使用方法，以及高级定制指南。

## 一、核心概念

在开始使用前，您需要了解几个核心概念：

- **Agentic Loop (自主工作流)**：当您给 AI 一个复杂任务时，它会自动分解任务，调用工具（如修改文件、执行测试），观察工具的输出，然后决定下一步该做什么。最大自动循环次数上限为 10 次。
- **MCP (Model Context Protocol)**：开放的工具接入协议。DolanClaw 可以通过 MCP 接入外部能力，比如查询数据库、搜索网页内容、自动控制浏览器等。
- **技能系统 (Skills & Agents)**：您可以将常用的业务背景或处理流程写成 Markdown 文件作为“技能”。AI 在遇到相关任务时会自动加载这些背景知识。

## 二、界面功能详解

平台左侧包含导航菜单栏，以下是所有可用功能面板的详细说明：

### 1. 💬 在线聊天 (Chat)
这是您最常使用的界面。
- **对话框**：直接向 AI 下达任务指令、询问代码逻辑或排查 bug。
- **模型切换**：对话框顶部可以随时切换后台用于推理的模型（支持 DeepSeek、MiniMax、Claude 等）。
- **工具调用展示**：当 AI 开始执行操作时，聊天流中会展示它具体调用了什么工具（如 `FileEditTool`，`BashTool`）及其结果。

### 2. 📊 仪表盘 (Dashboard)
了解当前会话与项目状态。
- **运行统计**：显示已使用的 Token 数量和大致 API 开销。
- **系统状态**：提示当前加载了哪些关键系统配置和环境信息。

### 3. 📂 文件浏览 (Files)
内置的树状文件浏览器。
- 可以展开查看项目下的文件结构。
- 支持语法高亮预览文件内容。
- *提示*：这是一个只读视图，用于协助您快速确认代码位置，实际代码修改交由 AI 自动完成。

### 4. 📝 代码差异 (Diff)
如果您使用了 Git 仓库，这里会展示当前代码的变更情况。
- **Staged / Unstaged**：清晰展示哪些文件被修改了但还没有提交。
- *提示*：在让 AI 提交代码前，可以先在这里检查改动是否正确。

### 5. 📋 任务看板 (Tasks)
Kanban 风格的任务管理界面。
- 会自动根据对话中识别出来的任务目标进行状态分类（如 Todo, In Progress, Done）。
- 您可以手动添加短期目标，AI 有时也可根据提示词帮您管理任务。

### 6. 🧠 项目记忆 (Memory)
管理不同维度的上下文记忆：
- **项目级**：当前工作目录的背景知识。
- **用户级**：您的个人偏好。
- **关联文档**：加载并解析出来的相关文档记忆。

### 7. 🔌 MCP 服务器 (MCP)
管理外部工具集成。
- 可以管理通过 `.mcp.json` 读取的服务器。
- 展示各个 MCP 服务器当前的状态和它们提供的具体功能列表。

### 8. 🧩 技能中心 (Skills) & 🤖 代理管理 (Agents)
- **技能 Tab**：展示了系统已发现的所有 `.claude/skills/` 下的 Markdown 文件。您可以开启或关闭某些自动触发的技能。
- **代理 Tab**：您可以查看各种细分领域专家 Agent，并可以通过这里配置它们使用的模型，或者单独唤起它们执行专有任务（例如唤起专门的代码审查 Agent）。

### 9. 🛠️ 工具浏览 (Tools)
查看当前 AI 可以调用的底层内置能力（如文件读写、搜索、Shell 终端）。可以让您直观体验某个工具输入参数和返回的结果。

### 10. 🏪 扩展市场 (Registry)
用于安装第三方扩展的一站式商店。
- **MCP 页面**：能一键安装像 GitHub 集成、Postgres 集成或 Playwright 等 MCP 插件。
- **技能包页面 (ECC 商店)**：集成了 Everything Claude Code 社区的精选扩展。提供完整的能力包（如“核心能力包”、“DevOps 包”）和独立组件。点击安装后会自动将其加入到本地的 `.claude/` 目录下。

## 三、如何与 AI 高效协作

1. **设定具体的可验证目标**：
   - ✖️ 不好："帮我优化这块代码"
   - ✔️ 更好："重构 `src/utils.ts` 中的 `calculate` 函数，提取公共逻辑。修改完后运行 `npm run test` 确保没有报错。"
2. **利用 Agentic 循环**：
   - 如果 AI 第一把尝试执行脚本失败了，请不要立刻接管。它通常会自动读取报错信息，通过下一轮循环来自行修复。
3. **快捷命令 (Slash Commands)**：
   - 输入 `/代码审查` 或定制的 `/xxx` 可以快速触发指定背景的流程。

## 四、扩展与定制指南 (高级)

DolanClaw 被设计为高度可定制。用户通过在工作区创建 `.claude` 目录来实现定制化。

### 💡 您【可以】且推荐定制的内容

1. **添加新的模型与 API Key**
   - 只需在项目根目录编辑 `.env` 文件。支持随意添加更多国内外大型语言模型的 API Key。
2. **添加自定义 MCP 服务器**
   - 在项目根目录创建 `.mcp.json` 文件并定义服务器命令，或直接在“扩展市场”中一键安装。
3. **编写个人专属“技能” (Skills)**
   - 在 `.claude/skills/` 目录下创建任意 `.md` 文件，使用 YAML frontmatter 标注 `name`, `description`, `trigger`。
   - 当 `trigger: auto` 时，如果您的提问与之相关，系统自动将文件内容灌入上下文！
4. **编写自定义行为“代理” (Agents)**
   - 在 `.claude/agents/` 下编写代理角色的描述。
5. **增设全局或项目“规则” (Rules)**
   - 在 `.claude/rules/` 下创建 Markdown 指定缩进、开发框架偏好（如：禁止使用 Tailwind，仅用 CSS Module 等）。
6. **配置工作流钩子 (Hooks) ⚠️(有风险)**
   - 可以在 `.claude/hooks.json` 中定义 bash 脚本钩子，在工具执行前后触发，甚至可以阻止高危操作。

### 🚫 您【绝不应该】修改的内容

1. **逆向打包后的系统核心 `$DOLANCLAW_ROOT/src/entrypoints/web.ts` 等文件**
   - 如果您让 AI 修改这里面的代码，系统会立即崩溃或无法再次启动。DolanClaw 是基于反编译产物构建的，随意修改底层逻辑十分危险。
2. **手动写入非文本类资产到核心目录**
   - 虽然有内置工具，但尽量避免使用 bash 命令在 DolanClaw 平台自己的源代码里动态修改运行时的状态结构。

### 本地组件自动发现机制

无论您是用 Git Clone 下载的扩展，还是通过扩展市场安装，或是手动把 `.md` 文件拖进来，只要放入对应的 `.claude/skills`、`.claude/agents` 或 `.claude/commands` 目录下，**DolanClaw 的扩展中心会自动发现它们，并在 UI 中展示。**

## 五、限制与安全建议

1. **文件修改权限**：DolanClaw 有能力彻底修改您的项目文件甚至删除文件。在进行大型重构前，**请务必确保用 Git 提交了代码**，以便在 AI 出错时安全回滚。
2. **Bash 执行权限**：AI 会代您执行各类终端命令。系统对可能阻断主进程执行的 Hook 施加了 **5秒超时保护**，但是 AI 自主调用的 Bash 进程依然具有您的本地电脑执行权限。
3. **隔离性建议**：如果是运行未知开源项目的构建指令，请保持谨慎，特别是当 AI 自主尝试下载和安装第三方包时。您可以为 DolanClaw 平台单独开启有文件结构限制的用户环境以提高安全性。
