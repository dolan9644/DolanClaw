# DolanClaw 完整使用指南 (V2.0)

本指南旨在帮助您从入门到精通使用 DolanClaw 平台。DolanClaw 不仅仅是一个聊天窗口，它是一个由 **Agentic Loop** 驱动的本地开发中枢。

---

## 一、核心概念

### 1. 什么是 Agentic AI？
普通的对话 AI 只能回答问题，而 **Agentic AI** 具备“自主权”。当您输入“帮我重构这个函数并跑通测试”时，AI 会经历：
1. **规划**：决定需要读取哪些文件。
2. **执行**：使用 `FileRead` 工具读取代码。
3. **思考**：分析逻辑并编写修改方案。
4. **行动**：使用 `FileEdit` 工具应用修改。
5. **验证**：使用 `Bash` 工具运行测试命令。
6. **迭代**：如果测试报错，它会根据错误信息自动进行下一轮修复。

### 2. 什么是 MCP (Model Context Protocol)？
MCP 是 Anthropic 发布的开放标准。它允许 AI 安全地突破文件系统，访问外部世界——例如搜索互联网、查询数据库、操作浏览器、访问 GitHub API 等。

## 二、界面功能详解

DolanClaw 的所有功能通过左侧导航栏访问：

- **💬 在线聊天 (Chat)**：主入口。支持多轮工具调用展示。
- **📊 仪表盘 (Dashboard)**：查看当前 Token 消耗监控和项目活跃度统计。
- **📂 文件浏览 (Files)**：实时查看项目目录，辅助定位文件位置。
- **📝 代码差异 (Diff)**：检查 AI 的改动。在 AI 提出提交代码前，建议在这里快速 Review。
- **📋 任务看板 (Tasks)**：将 AI 的目标物化。支持 Todo / In Progress / Done 三状态。
- **🧠 项目记忆 (Memory)**：AI 的持久化笔记本。它可以自动保存跨会话的上下文，包括技术栈偏好和常见错误总结。
- **🔌 MCP 管理 (MCP)**：查看已启用的工具服务器、连接状态及它们提供的数千个子工具。
- **🧩 技能中心 (Skills)**：管理您的业务逻辑模板和专家 Agent 的配置。
- **🛠️ 工具浏览器 (Tools)**：像 API 文档一样测试 AI 的所有底层动作。
- **🏪 扩展市场 (Registry)**：搜索并一键安装来自 Everything Claude Code 社区和 MCP 官方的扩展。

## 三、扩展功能指南 (极简/深度自定义)

这是 DolanClaw 最强大的地方。您可以根据项目需求无限扩展它的能力。

### 1. 哪些功能是可以扩展的？
| 扩展类型 | 存放位置 (工作区目录) | 说明 |
| :--- | :--- | :--- |
| **工作流技能 (Skills)** | `.claude/skills/` | Markdown 文件。定义特定的编码风格、背景知识或复杂任务模板。 |
| **专家代理 (Agents)** | `.claude/agents/` | Markdown 文件。定义具有特定身份的 Agent（如：安全审查员、SQL 专家）。 |
| **快捷命令 (Commands)** | `.claude/commands/` | Markdown 文件。定义 `/xxx` 这种斜杠命令及其预设 Prompt。 |
| **编码规则 (Rules)** | `.claude/rules/` | 定义项目级强制标准（如：禁止使用 Axios，必须使用 Fetch）。 |
| **生命周期钩子 (Hooks)** | `.claude/hooks.json` | 可扩展的 Bash 拦截器。支持工具调用前后、会话开始等 8 个生命周期点。 |
| **工具能力 (MCP)** | `.mcp.json` | 配置文件。定义外部工具服务器的二进制运行路径和参数。 |

---

### 2. 推荐安装与配置
为了获得最佳体验，我们建议首选安装以下内容：
- **🧠 核心能力包 (来自扩展市场)**：包含 ECC 社区最精良的 `planner` (规划者) 和 `code-reviewer` (代码审查) 代理。
- **MCP - Filesystem**：这是 AI 读写本地文件的基础。
- **MCP - Sequential Thinking**：让 AI 具备“深思熟虑”模式，在处理复杂逻辑时更不容易出错。
- **MCP - Memory**：在项目过大时，通过知识图谱帮助 AI 建立全局视野。

### 3. MCP 服务器深度介绍

#### (A) 如何添加 MCP 服务器？
目前支持两种方式：
1. **自动化安装**：在“扩展市场 (Registry)” -> “MCP 页面”选择需要的服务器，点击“安装”。如果需要 API Key，系统会弹出输入框。
2. **手动添加**：在您的项目根目录创建或编辑 `.mcp.json`。格式如下：

```json
{
  "mcpServers": {
    "my-custom-server": {
      "command": "npx",
      "args": ["-y", "@author/package-name", "参数1"],
      "env": {
        "API_KEY": "xxx"
      }
    }
  }
}
```

#### (B) 具体参数含义
- **服务器名称 (Key)**：例如上面的 `my-custom-server`。这仅是标识名，您可以随意填写，AI 在内部会通过这个名字来识别工具分类。
- **Command**：启动服务器的命令，通常是 `npx`, `node`, `python3` 或二进制文件路径。
- **Args**：传递给命令的参数。对于大多数基于 npm 的 MCP，通常是 `["-y", "@scope/server-name"]`。

### 4. 调用与协同工作机制

#### (A) 应该连接多少个 MCP？
- **推荐连接数：2 - 6 个**。
- **为什么？** 每个 MCP 都会暴露数个甚至数十个工具，这些工具的描述信息会占用 AI 的 **上下文窗口 (Context Window)**。连接过多（超过 10 个）会导致 AI 的推理能力下降，并浪费大量的 Token 成本。

#### (B) 它们是如何协同工作的？
DolanClaw 会在每一轮对话开始前，将所有已连接 MCP 服务器的工具列表“拼”进系统提示词。
- **自动分工**：AI 会根据您的需求自动判断。例如：“把这个页面的截图发到 Slack”，AI 会先调用 `puppeteer` 截图，再调用 `slack` 发送。它知道不同 MCP 服务器之间的输出可以作为对方的输入。
- **无感增强**：大部分时间您不需要手动指定使用哪个 MCP，直接说“帮我查一下 GitHub 上最新的 Issue”即可。

#### (C) 什么是“持续增加”的？
- **技能与 Agent 自由生长**：您的 `.claude/` 目录是可以随着项目演进不断增加内容的。只要您在对话中教育 AI 一些新的模式，并让它“把这个存为 Skill”，下次它就能直接复用。
- **热更新感知**：DolanClaw 具备本地文件监听和自动发现机制。您手动拖入一个 `.md` 技能文件后，无需重启，AI 会在下一次对话中自动感知。

## 四、安全与回滚建议

- **Git 是最后的防线**：在下达大规模修改指令前，务必 `git commit`。虽然 AI 提供了 Diff 视图，但如果循环次数过多，AI 可能会产生“幻觉”。
- **Hook 超时保护**：DolanClaw 内置了 **5 秒超时限制**。如果您的自定义 Hook 脚本死循环或长时间无响应，系统会强制 Kill 掉它并跳过，以保证主对话不卡死。
- **API Key 安全**：您的所有 API Key 均保存在本地 `.env` 或系统中。DolanClaw 核心程序禁止 Bot 修改任何核心运行逻辑（`src/entrypoints/web.ts` 等），确保您的 Key 不会被泄露。

## 五、卸载与清理 (Uninstallation & Cleanup)

DolanClaw 作为一个开源工具，不会在您的系统中强制“驻留”。如果您需要移除它，请参考以下步骤：

### 1. 移除软件主体
由于 DolanClaw 是通过 Git 克隆并在本地运行的，您只需**直接删除整个 `DolanClaw` 项目文件夹**即可完成主程序的“卸载”。

### 2. 清理全局配置与缓存
DolanClaw 在运行过程中可能会在您的用户目录下创建以下文件，如需彻底清理，请手动删除：
- `~/.claude/` - 包含全局设置（API Key 缓存等）和本地技能。
- `~/.mcp.json` - 如果您配置了全局 MCP 服务器。

### 3. 数据安全与原生技能保障 (重要)
- **原生技能保护**：如果您在 `.claude/skills/` 文件夹中手动放置了原生的 Claude Skills（`.md` 文件），DolanClaw 的“市场卸载”按钮**绝对不会**触碰这些文件。
- **选择性删除**：市场中的“卸载”功能仅根据安装清单删除对应的组件。它不会执行类似于 `rm -rf` 的清空操作，因此您的个人创作和原生习惯是受到物理隔离保护的。

## 六、如何更新 (How to Update)

DolanClaw 正在快速迭代中，建议您定期同步最新的功能和修复。

### 1. 更新主程序
由于本项目开源于 GitHub，最简单的方法是使用 Git 命令：
```bash
# 1. 确认已存盘并停止运行后端/前端
# 2. 拉取最新改动
git pull origin main

# 3. 更新依赖 (如果 package.json 发生了变动)
bun install
cd web && npm install && cd ..
```

### 2. 更新扩展市场 (Registry)
扩展市场的数据是动态同步的。如果您发现某个技能包显示有更新，通常只需在“扩展市场”中先点击“卸载”，然后再重新点击“安装”即可获取最新版本的文件。

### 3. 注意事项
- **冲突处理**：如果您手动修改过 DolanClaw 的核心代码（如 `web/` 下的代码），`git pull` 可能会报错。建议您先用 `git stash` 暂存您的改动，拉取后再恢复。
- **配置保留**：`.env` 和 `.mcp.json` 等配置文件通常是被 Git 忽略的，更新代码时**不会丢失**您的 API Key。
